## Slide 2 — 문제 정의 (0:10~0:50)

**제목**: 왜 이 문제인가?

- 🎵 Spotify에는 매일 6만 곡이 새로 올라온다
- 대부분의 곡은 **거의 재생되지 않는다**
- 아티스트/기획사: "어떤 음악이 뜰까?" → 실질적 가치

**우리의 질문**:
> "유명세·마케팅 없이, **소리 자체만**으로 인기곡을 구분할 수 있는가?"

**왜 실현 가능한가?**
→ Spotify는 모든 곡의 음향 특성을 자동으로 수치화한다 (발매 전도 계산 가능)

## Slide 3 — 데이터 소개 (0:50~1:20)

**제목**: 114,000곡의 소리를 수치로

**데이터**: Spotify Tracks Dataset (Kaggle, 공개 데이터)

| 피처 | 쉬운 설명 |
|------|----------|
| danceability | 이 노래에 춤출 수 있나요? (0~1) |
| energy | 얼마나 강렬한가요? (0~1) |
| loudness | 얼마나 큰 소리인가요? (dB) |
| instrumentalness | 보컬 없이 악기만? (0~1) |
| explicit | 선정적 가사가 있나요? (0/1) |
| ... | 총 13개 피처 사용 |

**타겟**: popularity ≥ 60이면 "인기곡" (전체의 약 11%)

> 💡 발견: 인기곡이 11%밖에 없다 → 클래스 불균형 문제 (뒤에서 설명)

---

## Slide 4 — 접근 방법 (1:20~2:00)

**제목**: 어떻게 풀었나 — 3단계 모델 비교

```
[단순] Logistic Regression  ─ baseline: 선형 경계만 학습
         ↓ 성능 부족 확인
[중간] Random Forest         ─ 수백 개 결정트리 앙상블
         ↓ 여전히 한계
[최종] XGBoost               ─ 실수를 고쳐가며 반복 학습
         + 하이퍼파라미터 자동 탐색 (60번 실험)
```

**중간에 발견한 문제**:
- 초기 실험에서 Accuracy 89%인데 F1 = **0.00**
- → 모델이 모든 곡을 "비인기"로 찍고 있었음
- → 해결: 인기곡에 8배 더 높은 가중치 부여

> 이게 바로 **정확도(Accuracy)만 믿으면 안 되는 이유**

---

## Slide 5 — 실험 결과 (2:00~2:50)

**제목**: 결과 — XGBoost가 가장 좋았다

| 모델 | F1 Score | AUC |
|------|----------|-----|
| Logistic Regression | 0.242 | 0.644 |
| Random Forest | 0.277 | 0.703 |
| **XGBoost (tuned)** | **0.294** | **0.722** |

**평가 지표 설명** (청중 배려):
- **F1 Score**: 인기곡을 얼마나 잘 찾아내는가 (0~1, 높을수록 좋음)
- **AUC**: 랜덤 추측(0.5)보다 얼마나 잘 구분하는가

> XGBoost가 랜덤 대비 22%p 더 정확하게 인기곡 구분

## Slide 6 — 무엇이 인기를 결정하나? (2:50~3:30)

**제목**: 가장 중요한 피처는?

[Feature Importance 그래프 삽입]

**Top 3 발견**:
1. 🥇 **explicit (선정적 가사)** — 가장 중요!
   → 힙합·R&B 장르의 강세 반영
2. 🥈 **instrumentalness (보컬 없음)** — 보컬 없으면 비인기

3. 🥉 **loudness (음량)** — 크고 강한 곡이 유리

**인사이트**: 인기곡은 "크고, 보컬 있고, 현대적인 장르"

## Slide 7 — 왜 어렵고 어디서 실패하나? (3:30~4:10)

**제목**: 한계와 실패 분석

**모델이 틀리는 경우**:
- popularity 45~65점 "애매한 곡" → 경계선 판단이 어려움
- 동일한 오디오 특성이어도 유명 아티스트 vs 신인 → 인기 천차만별

**근본적 한계**:
```
오디오 피처만으로는 설명 못하는 것들:
  아티스트 팬덤 크기
  마케팅 비용
  앨범 발매 시기
  소셜미디어 바이럴
```

→ 최고 F1 = 0.294: 랜덤보다는 훨씬 낫지만, 실서비스엔 부족
